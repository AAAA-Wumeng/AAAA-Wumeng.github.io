shell

基础

shell：一个可以解释shell规定的语法命令的解释器【解释器负责将应用程序发送的指令，进行检查，合法后交由内核执行！返回结果】【接收应用程序、用户命令，然后调用操作系统】

当应用程序在执行的时候，需要调用计算机硬件的CPU、内存等资源；程序将指令发送给内核执行

为了防止程序发送一些恶意指令导致损坏内核，在内核和应用程序之间设置一个中间层，称为shell。

shell命令：shell解释器要求的指定的语法编写的命令

shell脚本：多条shell命令，可以编写在一个文件中，文件中的指令，可以按照顺序来执行，将这个文件称为shell脚本。

Linux系统的核心是内核【kernal】。内核控制者计算机系统上的所有硬件和软件，在必要的时候为应用程序分配硬件，并根据需要执行程序中的代码；内核负责以下四种功能：

  • 系统内存管理
  • 软件程序管理
  • 硬件设备管理
  • 文件系统管理

Shell是一种特殊的交互式工具，核心是命令提示符，允许输入文本命令，解释命令，并在内核中执行命令。

shell中的解释器

查看Centos7中支持的解释器：

执行cat /etc/shells命令的在底层执行的是：/bin/bash -c ‘cat /etc/shells’

不同解释器能解释的命令不同。

/bin/bash是Centos默认的解释器，默认/bin/bash必须接一个脚本，作为输入！如果是一条命令，需要加-c参数【-c表示command】

/bin/bash：可以用来执行登录，也可以用来执行其他命令

/sbin/nologin：不可以用来执行登录，其他的都可以

查看Centos默认的解释器

使用$SHELL变量查看当前系统默认的解释器类型

Linux中的常用目录

  • bin：linux用户常用的命令【可以使用which搜索常用的命令在哪里】
  • sbin（super user bin）：超级管理员用的bin【root用户使用的常用命令】，对整个机器的管理命令

-bash: aaaa: command not found：表示当前命令不在当前用户的环境变量中【linux中多个环境变量用：间隔】

脚本的编写要求

  • 1、声明：**#/bin/bash**
  • 2、正文：必须是shell解释器能解释的命令

脚本的执行

脚本的执行方式

  • bash [脚本]
  • sh [脚本]
  • ./ [脚本]【./表示当前目录，不带./就从$PATH下的路径中找。】
    • 需要对脚本赋予执行的权限 - chmod u+x [脚本名称]【还原权限（chmod u-x [脚本名称]）】

  • source [脚本名称] 【source表示读取一个源文件，并执行源文件中的内容】
  • . [脚本名称]

脚本的编写要求

  • 声明：#!/bin/bash
  • 正文：必须使shell解释器能解释的命令
  • 脚本的执行 ~

脚本执行的特点

  • 1、bash /sh 脚本
    • 特点：新开一个bash执行脚本，一旦脚本执行完毕，bash自动关闭
  • 2、./ 脚本【前提是当前用户对脚本有执行权限】【使用的是默认的解释器执行脚本】
    • 特点：新开一个bash执行脚本，一旦脚本执行完毕，bash自动关闭
  • 3、source / . + 脚本【使用的是默认的解释器执行脚本】【并不要求当前用户对脚本有解释的权限】
    • 特点：在当前bash执行脚本

变量的操作

基本语法

  • 定义变量：变量=值
  • 撤销变量：unset 变量
  • 声明静态变量：readonly 变量，注意：不能使用unset

变量的定义规则

  • 变量的名称可以由字母数字下划线组成，但是不能以数字开头，环境变量名建议大写
  • 等号两侧不能有空格
  • 在bash中，变量默认的类型都是字符串类型，无法直接进行数值运算
  • 变量的值如果有空格，需要使用单引号【不能识别$等特殊字符】或者双引号【可以脱意$】括起来
  • 使用双引号，可以说识别空格之外的其他变量，使用单引号则会只会忽略空格！
  • 使用反引号，可以将语句运行的结果作为赋值给变量【例如将命令执行的结果，赋值给变量】
    • 命令等价于$(命令)类似的还有.跟一个脚本和source + 一个脚本

变量操作

  • 增： 变量名=变量值

  • 删： unset 变量名

  • 改： 变量名=变量值【用一个新的变量，赋值给变量就可以了】

  • 查： echo $变量名

    • 查看当前bash所有定义的变量：set

  • 特殊关键字：

    • readonly：用来修饰一个只读变量【不能写】
    • export：导出！将一个变量提升为全局变量！
      • 局部变量：默认变量只在定义的bash中有效，如果希望再bash-a中访问bash-b中定义的变量，要求：bash-b不能关闭；让bash-b将变量提升为全局变量，才能访问到

  • 变量的声明周期

    • 第一次新增的时候会产生
    • 变量会在执行unset时撤销
    • 关闭当前bash时，所有变量也不会再存在了【exit命令，可以关闭当前bash】

  • 特殊的变量

    • $?：表示上一条命令的返回值【echo $?为0，表示上一条命令执行成功，否则失败 】
    • $#：表示你传入的参数的个数
    • $*：和@在大部分情况下是一致的，在使用 ”$星“，时将整个参数列表作为第一个参数
    • $@：表示你的参数列表
    • $0~n：
      • $0：表示脚本名称
      • $1：表示你传入的第一个参数；同理$2，表示你传入的第二个参数；以此类推
      • 获取10以上的参数需要${n}才可以

运算符

基本语法

  • 整数：”$((运算式))”或者”$[运算式]”
  • expr +，-，*【需要加一个斜杠，因为斜杠在linux中表示一个通配符】，/，%，加，减，乘，除，取余
    • 运算符间，需要有空格
    • 使用反引号指定执行的顺序【例子：计算（1+1）* 2-1】

条件判断

基本语法

[ condition ]（注意condition之间要有空格）【注意条件非空即为true，是空的返回false】（0表示true，非0就是false）

**[ 1=0 ]**：返回true;

常用判断条件

（1）两个整数之间的比较

= 字符串比较；【左右两侧有空格，等号两侧也需要空格】

-lt 小于 less than；

-le 小于等于 less equal；

-eq 等于 （equal）

-gt 大于 （greater than）

-ge 大于等于 （greater equal）

-ne 不等于 （Not equal）

（2）按照文件权限判断【当前用户】

-r 有读的权限（read）；-w 有写的权限（write）；-x 有执行的权限（execute）

（3）按照文件类型进行判断

-f 文件存在并且是一个常规文件（file）

-e 文件存在（existence）

-d 文件存在并是一个目录

-s 文件存在且不为空【大小大于0】

-L 文件存在且是一个链接（link）

工具

wc：

用来计算数字，利用wc指令我们可以计算文件的Byte数，字数，或列数，若不指定文件名称，或是所给予的文件名为”-“，则wc指令会从标准输入设备读取数据。

基本用法：wc [选项参数] filename（不带选项参数，默认输出-l,-w,-c）

参数说明

选项参数 功能
-l 统计文件行数
-w 统计文件中的单词数
-m 统计文件中的字符数
-c 统计文件的字节数

cut：

在文件中负责剪切数据，cat命令从文件的每一行剪切字节、字符和字段并将这些字节、字符和字段输出

基本用法：cut [选项参数] filename（默认分隔符是制表符）

选项参数说明

选项参数 功能
-f f为fileds，列号，提取第几列
-d d为Descriptor分隔符，按照指定分隔符分割列

sed

基本用法：sed [选项参数] ‘command’ filename

选项参数说明：

选项参数 功能
-e 直接在指定列模式上进行sed的动作编辑

命令功能描述

命令 功能
a 新增，a的后面可以接字符串，在下一行输出
d 删除
s 查找并替换

sort

sort命令，将文件进行排序，并将排序的结果标准输出，默认情况以第一个字符出的字典顺序来排序

基本语法 sort (选项)（参数）

选项 说明
-n 按照数值的大小排序
-r 以相反的顺序来排序
-t 设置排序时所用的分割字符，默认使用TAB
-k 指定需要排序的列
-u u为unique的缩写，即如果出现相同的数据，只出现一行

参数：指定待排序的文件列表

awk

文本分析工具，把文件逐行读入，以空格为默认分隔将每行切片，切开的部分再进行分析处理

基本语法

awk[选项参数] ’pattern1{cation1} pattern2{action2}…‘ filename

pattern：表示AWK在数据中的查找内容，就是匹配模式

action：在找到匹配内容时，所执行的一系列命令

选项参数说明

选项参数 功能
-F 指定一个输入文件折分隔符
-v 赋值一个用户定义变量

关键字

  • print：向屏幕输出内容
  • $1-$n：第一列到第n列
  • BEGIN：代表开始读取文件之前做什么事，初始代码块
  • END：代表读取文件所有内容之后，做什么事，结束代码块

内置变量

变量 说明
FILENAME 文件名
NR 已读的记录数（行号）
NF 浏览记录的域的个数（切割后列的个数）
